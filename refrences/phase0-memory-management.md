# Memory Management System Design - Abjad Programming Language

**المرحلة:** ٠ - التأسيس (Foundation)
**الجزء:** ٣ - إدارة الذاكرة (Memory Management)
**التاريخ:** Q1 2025

---

## ١. نظرة عامة (Overview)

نظام إدارة الذاكرة في أبجد مستوحى من نظام Rust للملكية والاستعارة، مع تكييفه للغة العربية والبرمجة العربية. النظام يضمن سلامة الذاكرة دون الحاجة إلى جامع القمامة (garbage collector).

The memory management system in Abjad is inspired by Rust's ownership and borrowing system, adapted for Arabic programming. It ensures memory safety without a garbage collector.

---

## ٢. نظام الملكية (Ownership System)

### ٢.١ القواعد الأساسية (Basic Rules)

**القاعدة ١:** كل قيمة لها مالك واحد (Each value has a single owner)
```abjad
متغير owner = "قيمة"  // owner يملك القيمة
```

**القاعدة ٢:** عند انتقال الملكية، يصبح المالك السابق غير صالح (When ownership is transferred, the previous owner becomes invalid)
```abjad
متغير أ = "قيمة"
متغير ب = أ  // انتقال الملكية من أ إلى ب
// أ الآن غير صالح
```

**القاعدة ٣:** عند خروج المالك من النطاق، يتم تحرير الذاكرة (When the owner goes out of scope, memory is freed)
```abjad
{
    متغير قيمة = "بيانات"
}  // تحرير الذاكرة عند خروج القيمة من النطاق
```

### ٢.٢ أنواع الملكية (Ownership Types)

#### الملكية الكاملة (Full Ownership)
```abjad
متغير بيانات = [١، ٢، ٣]  // بيانات يملك المصفوفة بالكامل
```

#### الملكية المشتركة (Shared Ownership)
```abjad
// استخدام Arc للملكية المشتركة بين خيوط متعددة
استورد "أبجد/تزامن"

متغير بيانات_مشتركة = Arc::جديد([١، ٢، ٣])
```

#### الملكية المتبادلة (Swappable Ownership)
```abjad
// يمكن تبديل الملكية بين متغيرين
متغير أ = "قيمة"
متغير ب = "قيمة_أخرى"
أ، ب = ب، أ  // تبديل الملكية
```

### ٢.٣ انتقال الملكية (Ownership Transfer)

#### انتقال الملكية في التخصيص (Assignment)
```abjad
متغير أ = "قيمة"
متغير ب = أ  // انتقال الملكية
// أ الآن غير صالح
```

#### انتقال الملكية في الدوال (Function Calls)
```abjad
دالة تأخذ_الملكية(بيانات: نص) {
    // البيانات تنتقل إلى الدالة
}

متغير بيانات = "قيمة"
تأخذ_الملكية(بيانات)  // انتقال الملكية
// بيانات الآن غير صالحة
```

#### انتقال الملكية في الهياكل (Structs)
```abjad
هيكل النقطة {
    س: عشري٦٤،
    ص: عشري٦٤
}

متغير نقطة = النقطة { س: ١.٠، ص: ٢.٠ }
متغير نقطة_أخرى = نقطة  // انتقال الملكية
// نقطة الآن غير صالحة
```

### ٢.٤ استنساخ الملكية (Cloning Ownership)

#### الاستنساخ العميق (Deep Clone)
```abjad
متغير أ = [١، ٢، ٣]
متغير ب = أ.استنساخ()  // استنساخ عميق
// كلا المتغيرين صالحان
```

#### الاستنساخ السطحي (Shallow Clone)
```abjad
متغير أ = [١، ٢، ٣]
متغير ب = أ.استنساخ_سطحي()  // استنساخ سطحي
// كلا المتغيرين يشيران إلى نفس البيانات
```

## ٣. نظام الاستعارة (Borrowing System)

### ٣.١ قواعد الاستعارة (Borrowing Rules)

**القاعدة ١:** يمكنك استعارة قيمة واحدة عدة مرات (You can have multiple immutable borrows)
```abjad
متغير قيمة = "بيانات"
استعارة_ثابتة_١ = &قيمة
استعارة_ثابتة_٢ = &قيمة
// كلا الاستعارتين صالحتان
```

**القاعدة ٢:** يمكنك استعارة قيمة واحدة متغيرة مرة واحدة (You can have one mutable borrow)
```abjad
متغير قيمة = "بيانات"
استعارة_متغيرة = &متغير قيمة
// استعارة واحدة متغيرة صالحة
```

**القاعدة ٣:** لا يمكنك الجمع بين استعارات ثابتة ومتغيرة (You cannot mix immutable and mutable borrows)
```abjad
متغير قيمة = "بيانات"
استعارة_ثابتة = &قيمة
استعارة_متغيرة = &متغير قيمة  // خطأ
```

### ٣.٢ أنواع الاستعارة (Borrow Types)

#### الاستعارة الثابتة (Immutable Borrow)
```abjad
متغير قيمة = "بيانات"
استعارة = &قيمة  // استعارة ثابتة
// لا يمكن تعديل القيمة عبر الاستعارة
```

#### الاستعارة المتغيرة (Mutable Borrow)
```abjad
متغير قيمة = "بيانات"
استعارة = &متغير قيمة  // استعارة متغيرة
// يمكن تعديل القيمة عبر الاستعارة
```

#### الاستعارة المتعددة (Multiple Borrows)
```abjad
متغير قيمة = "بيانات"
استعارة_١ = &قيمة
استعارة_٢ = &قيمة
استعارة_٣ = &قيمة
// استعارات ثابتة متعددة صالحة
```

### ٣.٣ الاستعارة في الدوال (Borrowing in Functions)

#### استعارة ثابتة كمعامل (Immutable Borrow as Parameter)
```abjad
دالة اقرأ(بيانات: &نص) {
    طباعة(بيانات)
}

متغير قيمة = "بيانات"
اقرأ(&قيمة)  // استعارة ثابتة
// قيمة لا تزال صالحة
```

#### استعارة متغيرة كمعامل (Mutable Borrow as Parameter)
```abjad
دالة عدل(بيانات: &متغير نص) {
    *بيانات = "قيمة_جديدة"
}

متغير قيمة = "بيانات"
عدل(&متغير قيمة)  // استعارة متغيرة
// قيمة لا تزال صالحة
```

## ٤. نظام العمر (Lifetimes System)

### ٤.١ تعريف العمر (Lifetime Definition)

العمر هو النطاق الذي تكون فيه المرجع صالحاً.

A lifetime is the scope in which a reference is valid.

```abjad
دالة مثال<'عمر>(بيانات: &'عمر نص) -> &'عمر نص {
    // 'عمر يحدد عمر المرجع
    بيانات
}
```

### ٤.٢ قواعد العمر (Lifetime Rules)

**القاعدة ١:** عمر المرجع يجب أن يكون أقل من أو يساوي عمر المالك (Reference lifetime must be <= owner lifetime)
```abjad
{
    متغير مالك = "قيمة"
    استعارة = &مالك  // عمر الاستعارة <= عمر المالك
}
// تحرير المالك، تحرير الاستعارة
```

**القاعدة ٢:** عمر الإرجاع يجب أن يكون مرتبطاً بمدخلات الدالة (Return lifetime must be tied to function inputs)
```abjad
دالة أطول_سلسلة<'عمر>(أ: &'عمر نص، ب: &'عمر نص) -> &'عمر نص {
    إذا (أ.طول() > ب.طول()) {
        أ
    } آخر {
        ب
    }
}
```

### ٤.٣ الاستدلال التلقائي للعمر (Lifetime Elision)

#### القواعد (Rules)
1. كل معامل مرجع يحصل على عمر منفصل (Each reference parameter gets its own lifetime)
2. إذا كان هناك معامل مرجع واحد، يتم تعيينه لعمر الإرجاع (If there's one reference parameter, it's assigned to return)
3. إذا كان هناك معاملات مرجع متعددة، يجب تحديد العمر صراحة (If there are multiple reference parameters, must specify lifetime explicitly)

#### أمثلة (Examples)
```abjad
// الاستدلال التلقائي
دالة مثال(بيانات: &نص) -> &نص {
    بيانات
}

// تحديد صريح
دالة مثال<'عمر>(أ: &'عمر نص، ب: &'عمر نص) -> &'عمر نص {
    إذا (أ.طول() > ب.طول()) {
        أ
    } آخر {
        ب
    }
}
```

### ٤.٤ العمر الثابت (Static Lifetime)

```abjad
ثابت قيمة_ثابتة: &'static نص = "قيمة ثابتة"
// القيمة موجودة طوال عمر البرنامج
```

## ٥. تصميم البناء (Compiler Design)

### ٥.١ Borrow Checker

#### المسؤوليات (Responsibilities)
- التحقق من قواعد الاستعارة
- التحقق من قواعد العمر
- الإبلاغ عن أخطاء الاستعارة
- تحديد نطاق المتغيرات

#### الخوارزمية (Algorithm)
```
لكل متغير في النطاق:
    1. تحديد عمر المتغير
    2. تتبع الاستعارات
    3. التحقق من القواعد:
       - لا استعارات متغيرة مع استعارات ثابتة
       - عمر المرجع <= عمر المالك
       - عمر الإرجاع مرتبط بالمدخلات
```

### ٥.٢ تحليل النطاق (Scope Analysis)

#### أنواع النطاقات (Scope Types)
```abjad
// نطاق دالة
دالة مثال() {
    متغير محلي = "قيمة"
    // نطاق محلي
}

// نطاق حلقة
لكل رقم في ٠..١٠ {
    متغير حلقة = رقم
    // نطاق حلقة
}

// نطاق هيكل
هيكل المثال {
    قيمة: نص
}
// نطاق هيكل
```

### ٥.٣ تتبع الاستعارات (Borrow Tracking)

#### هيكل البيانات (Data Structure)
```rust
struct BorrowInfo {
    borrower: String,
    borrow_type: BorrowType,
    lifetime: Lifetime,
    scope: Scope,
}

enum BorrowType {
    Immutable,
    Mutable,
}
```

## ٦. تخصيص الذاكرة (Memory Allocation)

### ٦.١ تخصيص المكدس (Stack Allocation)

#### متى يستخدم (When to use)
- الأنواع ذات الحجم الثابت (Fixed-size types)
- المتغيرات المحلية (Local variables)
- المعاملات الدالة (Function parameters)

#### أمثلة (Examples)
```abjad
دالة مثال() {
    متغير عدد = ١٠  // تخصيص المكدس
    متغير نقطة = النقطة { س: ١.٠، ص: ٢.٠ }  // تخصيص المكدس
}
```

### ٦.٢ تخصيص الكومة (Heap Allocation)

#### متى يستخدم (When to use)
- الأنواع ذات الحجم المتغير (Variable-size types)
- المصفوفات الديناميكية (Dynamic arrays)
- الهياكل الكبيرة (Large structs)
- البيانات المشتركة (Shared data)

#### أمثلة (Examples)
```abjad
استورد "أبجد/مجموعة"

دالة مثال() {
    متغير مصفوفة = مجموعة::جديد()  // تخصيص الكومة
    مصفوفة.أضف(١)
    مصفوفة.أضف(٢)
}
```

### ٦.٣ تخطيط الذاكرة (Memory Layout)

#### تخطيط المكدس (Stack Layout)
```
┌─────────────────┐
│  متغيرات محلية  │
├─────────────────┤
│  معاملات دالة   │
├─────────────────┤
│  عنوان الإرجاع  │
└─────────────────┘
```

#### تخطيط الكومة (Heap Layout)
```
┌─────────────────┐
│  بيانات ديناميكية │
├─────────────────┤
│  بيانات مشتركة   │
├─────────────────┤
│  بيانات كبيرة    │
└─────────────────┘
```

## ٧. سلامة الذاكرة (Memory Safety)

### ٧.١ ضمانات (Guarantees)

#### لا تخصيص مزدوج (No Double Allocation)
```abjad
// ممنوع: تخصيص نفس الذاكرة مرتين
متغير أ = "قيمة"
متغير ب = أ  // انتقال الملكية، لا تخصيص مزدوج
```

#### لا تسرب ذاكرة (No Memory Leaks)
```abjad
// النظام يحرر الذاكرة تلقائياً عند خروج المالك من النطاق
{
    متغير بيانات = "قيمة"
}  // تحرير تلقائي
```

#### لا مؤشرات معلقة (No Dangling Pointers)
```abjad
// Borrow Checker يمنع المؤشرات المعلقة
متغير قيمة = "بيانات"
استعارة = &قيمة
// قيمة لا يمكن تحريرها بينما الاستعارة صالحة
```

### ٧.٢ التحقق من السلامة (Safety Checks)

#### في وقت التجميع (Compile-Time)
- التحقق من قواعد الاستعارة
- التحقق من قواعد العمر
- التحقق من انتقال الملكية الصحيح

#### في وقت التشغيل (Runtime)
- فحص الحدود (Bounds checking)
- فحص الذاكرة (Memory checking)
- فحص المؤشرات (Pointer checking)

## ٨. أمثلة عملية (Practical Examples)

### ٨.١ مثال بسيط (Simple Example)
```abjad
دالة رئيسية() {
    متغير اسم = "أحمد"
    طباعة(اسم)
}
```

### ٨.٢ مثال مع استعارة (Example with Borrowing)
```abjad
دالة اطبع_الاسم(اسم: &نص) {
    طباعة(اسم)
}

دالة رئيسي() {
    متغير اسم = "أحمد"
    اطبع_الاسم(&اسم)  // استعارة ثابتة
    طباعة(اسم)  // اسم لا يزال صالح
}
```

### ٨.٣ مثال مع انتقال ملكية (Example with Ownership Transfer)
```abjad
دالة تأخذ_الاسم(اسم: نص) {
    طباعة(اسم)
}

دالة رئيسي() {
    متغير اسم = "أحمد"
    تأخذ_الاسم(اسم)  // انتقال الملكية
    // اسم الآن غير صالح
}
```

### ٨.٤ مثال مع عمر (Example with Lifetime)
```abjad
دالة أطول_سلسلة<'عمر>(أ: &'عمر نص، ب: &'عمر نص) -> &'عمر نص {
    إذا (أ.طول() > ب.طول()) {
        أ
    } آخر {
        b
    }
}

دالة رئيسي() {
    متغير سلسلة_١ = "سلسلة طويلة"
    متغير سلسلة_٢ = "قصيرة"
    متغير أطول = أطول_سلسلة(&سلسلة_١، &سلسلة_٢)
    طباعة(أطول)
}
```
